任务说明

使用hexo和Github Page实现个人博客的搭建

前置要求

* 开发机器和Github帐号已经实现SSH公钥关联

工具安装以及创建博客项目

说明:

  • hexo-cli, hexo提供的命令行工具, 博客站点的生成和部署的相关操作, 都由该命令来完成
  • hexo-deployer-git: github部署器, 通过配置可以实现本地到github的直接部署

生成后的目录树大致如下:

.
├── _config.yml
├── package.json
├── scaffolds
│   ├── draft.md
│   ├── page.md
│   └── post.md
├── source
│   └── _posts
├── themes
│   └── landscape
└── yarn.lock

说明:

  • _config.yml: 项目配置主文件, 与github page仓库的部署设置就是在这里, 里面由很多预生成的条目
  • scaffolds/ 目录: 写成的文章, 草稿, 页面都是依照这里的骨架文件
  • source/_posts/: 在hexo generate操作之前, 将博客文章放在这里
  • themes/: 主题目录, 放置用户的博客主题, landscape是hexo默认主题

新主题安装

这里选用著名的next主题. 保持在项目根目录下

安装结果, themes目录下多了一个next目录:

.
├── landscape
└── next

主题选用以及Github page仓库绑定

这两个步骤都是通过配置项目根目录_config.yml来实现的

主题选泽

打开_config.yml文件, 定位到theme条目, 注释默认配置, 并依照形式, 添加对next主题的启用

theme: next

部署设置

同样在_config.yml文件, 定位到deploy条目, 添加两个主要信息, 结果如下:

deploy:
type: git
repo: git@github.com:lewenweijia/lewenweijia.github.io.git

当然, 其中的repo信息应该替换成你的Github page仓库地址.
注意, 使用git协议而不是https协议, 不然的话每次部署都得输入密码. 使用ssh密钥的方式可以实现免密码验证.

构建和部署

hexo init 命令会生成一个初始的post, 我们借助这个来看最终的部署效果

source/_posts/
├── hello-world.md

构建public目录

命令完成后, 根目录会生成新增一个public文件. 后续的部署任务便是将该文件下的所有资源部署到博客平台.

部署到GitHub Page上

由于我们之前已经与对应的Github Page 仓库已经实现关联, 就可以直接将public进行推送了

顺利的话, Github上已经有了推送结果, 试着打开你的github page网站, 开始hexo博客之旅吧

结语

折腾了小半下午, 总算把hexo搭建静态博客站点的工作流摸顺, 接下来一些相关概念慢慢接触学习并投入使用.

缺失模块。
1、请确保node版本大于6.2
2、在博客根目录（注意不是archer根目录）执行以下命令：
npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: true
    tags: true
